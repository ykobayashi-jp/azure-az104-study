AZ-104 頻出サービスの要点一覧

Microsoft Entra ID (Azure AD)
クラウドIDプラットフォーム。ユーザー/グループ/アプリ管理・SSO・MFAを提供。Free/P1/P2の3ライセンス。
ID管理
RBAC（ロールベースアクセス制御）
Owner（全権）/ Contributor（作成・変更）/ Reader（閲覧）/ UAA（ロール割り当て）。スコープはMG→サブスク→RG→リソース。
アクセス制御
Managed Identity
AzureリソースにAzure ADのIDを自動付与。System-assigned（リソース紐付き）/ User-assigned（共有可）。資格情報の管理不要。
セキュリティ
Conditional Access
条件ベースのアクセス制御。ユーザー・デバイス・場所・アプリを条件にMFA要求・アクセスブロックを自動適用。P1以上で利用可。
P1
PIM（Privileged Identity Management）
Just-In-Timeな特権ロール管理。Eligible（適格）→アクティブ化（承認・期限付き）。審査・監査ログも提供。P2必須。
P2
Azure AD B2B
外部ユーザーをゲストとして招待。取引先は自社IDでサインイン。アクセス権はRBAC/Conditional Accessで管理。
外部ID
Azure AD Connect
オンプレミスAD DSとEntra IDを同期。Password Hash Sync / Pass-through Auth / Federation(ADFS)の3認証方式。
ハイブリッド
SSPR（セルフサービスパスワードリセット）
ユーザー自身がパスワードをリセット。ヘルプデスク負荷軽減。P1以上またはMicrosoft 365で利用可。
P1
Azure Virtual Machines
IaaS仮想マシン。サイズはB（バースト）/ D（汎用）/ E（メモリ）/ F（CPU）/ N（GPU）シリーズ。Availability Sets / Zonesで可用性向上。
IaaS
VM Scale Sets (VMSS)
同一VMを複数管理・自動スケール。メトリクス/スケジュールベースのスケールイン・アウト。最大1000インスタンス。
自動スケール
Azure App Service
PaaS Webアプリホスティング。.NET/Python/Node/Java対応。Deployment Slots（スロット）でゼロダウンタイムデプロイ。
PaaS
Azure Functions
サーバーレス関数。イベントトリガー（Blob/Queue/Timer/HTTP）。従量課金（実行時間・回数）/ Premium / App Service Plan。
サーバーレス
AKS（Azure Kubernetes Service）
マネージドK8s。コントロールプレーンはAzure管理。HPA（水平スケール）・Node Pools・AGIC統合。
コンテナ
Azure Container Instances
サーバーレスコンテナ実行。秒単位課金・高速起動。バッチ処理・CI/CDエージェントに最適。
コンテナ
Azure Blob Storage
オブジェクトストレージ。Hot/Cool/Archive の3層。ライフサイクル管理で自動移行。ブロック/追記/ページブロブの3種類。
オブジェクト
Azure Files
フルマネージドSMB/NFSファイル共有。Azure File Syncでオンプレキャッシュ。Standard/Premiumの2ティア。
ファイル共有
Azure Disk（Managed Disk）
VM用ブロックストレージ。Ultra/Premium SSD v2/Premium SSD/Standard SSDの4種。ゾーン冗長・スナップショット対応。
ブロック
ストレージ冗長性
LRS（3コピー同一DC）/ ZRS（3AZ）/ GRS（セカンダリリージョン）/ GZRS（ZRS+GRS）。読み取りアクセスはRA-GRS/RA-GZRS。
冗長性
SAS（Shared Access Signature）
ストレージへの委任アクセス。有効期限・権限・IP制限を設定。Account SAS / Service SAS / User Delegation SASの3種。
アクセス制御
Azure Import/Export
物理ディスクでの大量データ転送。TB〜PBスケール。ハードディスクをAzureデータセンターに送付してインポート。
データ移行
Azure Virtual Network (VNet)
AzureのプライベートネットワークIPアドレス空間。サブネット・NSG・ルートテーブルで構成。リージョン単位で作成。
ネットワーク
NSG（ネットワークセキュリティグループ）
L3-L4のステートフルフィルタリング。サブネットまたはNICに適用。優先度順にルール評価。デフォルトルールで同VNetからの通信を許可。
セキュリティ
VNet Peering
2つのVNet間をMicrosoftバックボーンで接続。低レイテンシー・高帯域。リージョン内（Local）/ リージョン間（Global）の2種。
VNet接続
Application Gateway
L7ロードバランサー。URLパス/ホストベースルーティング・SSL終端・WAF機能。リージョン単位のサービス。
L7 LB
Azure Load Balancer
L4（TCP/UDP）ロードバランサー。Basic（無料・フィーチャー限定）/ Standard（ゾーン冗長・SLA 99.99%）。パブリック/内部の2種。
L4 LB
Private Endpoint
PaaSサービス（Storage/SQL等）へのプライベートIP接続。インターネットを経由しない。Private DNS Zoneで名前解決。
プライベート接続
Azure VPN Gateway
オンプレミスとのサイト間VPN（IPSec/IKE）接続。Basic/VpnGw1〜5のSKU。最大10Gbps（VpnGw5）。
ハイブリッド
ExpressRoute
専用線接続（インターネット不使用）。最大100Gbps。プロバイダー経由またはExpressRoute Direct。Global Reachで拠点間接続。
専用接続
Azure DNS
パブリックDNSゾーン（独自ドメイン管理）/ プライベートDNSゾーン（VNet内名前解決）の2種。Private Endpointと組み合わせて使用。
DNS
Azure Monitor
Azureリソースのメトリクス・ログ統合監視基盤。Metrics / Logs / Alerts / Dashboards / Workbooksで構成。
監視
Log Analytics ワークスペース
ログ収集・分析の中央ハブ。KQL（Kusto Query Language）で検索・集計。Azure Monitor Agentでデータ収集。保持期間設定可。
ログ分析
Application Insights
アプリのAPM。リクエスト・依存関係・例外・パフォーマンスを追跡。アプリケーションマップ・ライブメトリクス。Log Analyticsのワークスペースベース。
APM
Azure Alerts
メトリクス・ログ・アクティビティログを条件にアラート。Action Group（メール/SMS/Webhook/Runbook/Logic Apps）で通知・自動化。
アラート
Azure Policy
リソースに対してルールを強制。Deny/Audit/DeployIfNotExists/Modify等のエフェクト。Initiative（複数ポリシーのセット）で大規模管理。
ガバナンス
Azure Cost Management
コストの分析・予測・予算アラート。タグを使ったコストの部門別割り当て。Azure Advisorの推奨事項と連携。
コスト管理
Azure Backup
VM / SQL / SAP HANA / Blob / Azure Files / オンプレミス（MARS）をバックアップ。Recovery Services Vaultに保存。GRS/LRS選択可。
バックアップ
Recovery Services Vault
バックアップとSite Recoveryのデータ保管庫。ストレージ冗長性はGRS（デフォルト）/LRS/ZRS。Cross Region Restoreに対応（GRS時）。
保管庫
Azure Site Recovery (ASR)
VMの継続レプリケーションによるDR。RPO秒単位・RTO分単位。Azure→Azure / オンプレ→Azure / Azure→オンプレをサポート。
DR
MARS エージェント
オンプレミスWindowsサーバーのファイル/フォルダー/システム状態をAzure Backupへ直接バックアップ。エージェントインストールのみで設定完了。
オンプレバックアップ
Azure Backup ポリシー
バックアップのスケジュール（毎日/毎週）・保持期間（日次/週次/月次/年次）を設定。Enhanced Policyは時間単位バックアップに対応。
ポリシー

分野別ポイント

試験で狙われるポイントと落とし穴

ID管理（Identity）
  • 🔑
    PIMに必要なライセンスはAzure AD P2（Microsoft 365 E5）。Conditional AccessはP1でも可。
  • 💡
    Managed Identity（System-assigned）はリソース削除で自動削除。User-assignedは独立したライフサイクルで複数リソースに割り当て可能。
  • ⚠
    サービスプリンシパルのクライアントシークレットは有効期限があり、手動ローテーションが必要。Managed Identityを使えばこの管理が不要。
  • 🔑
    RBACの継承はMG→サブスクリプション→リソースグループ→リソースの順。上位スコープのロールは下位に継承される。
  • ⚠
    B2B vs B2Cの混同注意。B2Bは企業間コラボ（取引先ゲスト）、B2CはコンシューマーアプリのIDプラットフォーム（SNSログイン等）。
ストレージ
  • 🔑
    Archive層のリハイドレートは数時間かかる（高優先度で1〜3時間、標準で最大15時間）。アクセス頻度を正確に見積もること。
  • ⚠
    SASのアクセスキー共有はストレージアカウント全体のフルアクセス。外部共有にはSASを使い、必ず期限・権限・IP制限を設定する。
  • 💡
    Azure File SyncのCloud Tiering（クラウド階層化）は、よく使うファイルをオンプレに保持、残りはAzure Filesに保存してオンプレのディスク節約が可能。
  • 🔑
    GZRS vs GRS：リージョン障害+ゾーン障害の両方に備えるにはGZRS。リージョン障害のみならGRS（コスト低）。
コンピューティング
  • 🔑
    Availability Sets vs Availability Zones：Setsは同一DC内の障害ドメイン分散（ハードウェア障害対策）。Zonesは異なる物理ロケーション（ゾーン障害対策）でよりリスク分散が高い。
  • ⚠
    VMSSのスケールイン保護はインスタンスがスケールインで削除されないよう保護する機能。ステートフルなワークロードで重要。
  • 💡
    App Service Deployment SlotsはStandardプラン以上で利用可能。スワップ時にウォームアップが行われてから切り替わるため、ダウンタイムゼロ。
  • 🔑
    Azure Functions のトリガー：Blob（ファイル変更）/ Queue（メッセージ）/ Timer（スケジュール）/ HTTP（APIエンドポイント）/ Service Bus など多数。
ネットワーク
  • 🔑
    Service Endpoint vs Private Endpoint：Service Endpointはルーティング最適化（パブリックIPが残る）。Private EndpointはVNet内のプライベートIPでサービスにアクセス（インターネット経路なし）。
  • ⚠
    VNet Peeringは非推移的。A-BペアリングとB-CペアリングがあってもAからCへは通信できない。AとCを直接ペアリングするか、Azure Virtual WAN / NVAを使う。
  • 💡
    NSGの評価順：優先度の低い数値（100等）が先に評価される。同じ優先度の場合はDenyが優先。デフォルトルールはDenyAll（65500）。
  • 🔑
    Application GatewayとFront Doorの違い：Application Gatewayはリージョン単位、Front Doorはグローバル（Anycastで最近のPoPへルーティング）。
監視・ガバナンス
  • 🔑
    Azure Monitor vs Application Insights：Monitorはインフラ全体のメトリクス/ログ基盤。App Insightsはアプリのトレーシング・依存関係特化のAPMツール（Log Analyticsのワークスペースベース）。
  • 💡
    Azure Policy の DeployIfNotExists エフェクトは、条件を満たさないリソースに対して自動的にリソース（診断設定等）をデプロイする。Auditは検出のみ。
  • ⚠
    アクティビティログ vs リソースログ：アクティビティログはサブスクリプションレベルの操作（誰が何をしたか）。リソースログは個別リソースの動作ログ（Log Analyticsへ転送が必要）。
アーキテクチャ図解

AZ-104 頻出のアーキテクチャパターン

ハブ＆スポーク VNet トポロジー

Hub-Spokeパターン：共有サービス（Firewall・GW・Bastion）をHub VNetに集約し、各SpokeからVNet Peeringで接続。Spoke同士は直接通信せずHubを経由することでセキュリティポリシーを一元管理。

プライベートエンドポイントによるセキュアなPaaSアクセス

Private Endpointでサービスをプライベート化。DNSゾーンが自動的にプライベートIPに名前解決し、インターネットを経由しない安全な接続を実現。

DR構成（Azure Site Recovery）

ASRは継続的レプリケーションでRPOを最小化。障害時はフェイルオーバーでセカンダリのVMを起動。Recovery Planで複数VMの起動順序・スクリプトを自動化。
